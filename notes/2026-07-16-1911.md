وش يعني؟ أنا قاعدة أفكر في الـ REST — يعني Representational State Transfer. هذا مو بس style للـ APIs، هذا أساس تطوير back-end فعلاً.

في بالي إني شايفة ستة مبادئ وايد مهمة: أول حاجة الـ Idempotency، يعني الطلب نفسه دايم يرجع نتيجة نفسها. بعدين الـ Cache-Control — كيف المتصفحات تخزّن الـ resources. والـ Content Negotiation — أنهي format نرجع. الـ HATEOAS قاعدة تخليك تحط لينكات في الـ responses. الـ Conditional Requests مثل if-match و if-none-match. والـ Hypermedia — البيانات المربوطة ببعضها.

هالمبادئ هي اللي تعطيك structure للأنظمة اللي تبنينها على HTTP.

مو كويس بالنسبة لي إن كل framework — Rails، Phoenix، Express.js، Spring Boot — عنده طريقته الخاصة. ما فيه standardization واحد عالمي للـ مبادئ هذي مثل ما الـ REST نفسه.

REST يعطيني confidence إن الـ API بيسوي اللي يقول عنه، ورا وين ما يكون مرفوع.

بعدين صارت مصيبة — الـ GitHub token انتهى، وما بيت الأنتظار. رحت على قناة ثانية عندي وقت — publishing مباشر عن طريق Cloudflare بدون subdomain ولا token شيء يدوي.

في أداة الـ live | أنا فهد، والـ test خلصت، ما فيه حاجة مكتوبة عن الـ file.

الحقيقة، الـ REST اعطاني clarity بالعملي فقط. بس النظري؟ ما زين مُحقق عالمياً زي ما هو محقق محلياً بـ HTTP methods.

وهذا اللي مهم وايد عندي — مبدأ الـ HTTP methods في الـ REST بالذات كيف ينطبق على شنو نسويه.
